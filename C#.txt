1. What is C# and what are its key features?
C# is a modern, object-oriented, type-safe programming language developed by Microsoft as part of the .NET ecosystem. It is widely used for building Windows desktop apps, web apps (ASP.NET), mobile apps (Xamarin/MAUI), cloud services (Azure), games (Unity), and more.
Key features:

Strong typing with excellent IntelliSense
Object-oriented with support for functional programming (lambdas, LINQ)
Automatic memory management via Garbage Collector
Cross-platform with .NET (formerly .NET Core)
Rich async support (async/await)
Modern features: records, pattern matching, nullable reference types, source generators, etc.
High performance with Span<T>, ref structs, and AOT compilation

3. What are the different types of data types available in C#?
C# has two main categories:

Value Types: Stored on stack (or inline), copied by value. Examples: int, float, double, bool, char, struct, enum, decimal.
Reference Types: Stored on heap, copied by reference. Examples: class, string, array, delegate, object, interface.

Also: Nullable value types (int?), dynamic, and pointer types (unsafe context).
4. What is the difference between value types and reference types?

Value Types: Contain actual data. Assignment copies the value. Passed by value to methods. Live on stack (unless boxed). Examples: primitives, structs.
Reference Types: Contain a reference (pointer) to the data on the heap. Assignment copies the reference. Passed by reference. Can be null.

Example:
C#int a = 10;          // value type
int b = a;           // b is copy of 10
b = 20;              // a remains 10

string s1 = "hello"; // reference type
string s2 = s1;      // s2 points to same string
s2 = "world";        // s1 still "hello" (strings are immutable)
5. What are nullable types in C#?
Nullable types allow value types to hold null. Introduced in C# 2.0 as Nullable<T>, shorthand T?.
C#int? age = null;
if (age.HasValue)
    Console.WriteLine(age.Value);
else
    Console.WriteLine("Age not set");

// C# 8+ Nullable Reference Types (for reference types)
string? name = null;   // Explicitly nullable
Helps prevent NullReferenceException with compiler warnings.

7. Explain the concept of boxing and unboxing in C#.

Boxing: Converting a value type to a reference type (object). Allocates memory on heap.

C#int i = 10;
object obj = i;   // boxing (performance cost)

Unboxing: Converting reference type back to value type.

C#int j = (int)obj; // unboxing
Boxing/unboxing has performance overhead. Avoid in hot paths; use generics instead.
8. What is Type Casting and what are its types in C#?
Type casting converts one type to another.

Implicit Casting: Automatic, safe (e.g., int to long).
Explicit Casting: Manual with (Type) — can throw InvalidCastException.
as operator: Safe casting for reference types (returns null on failure).
is operator: Checks compatibility before casting.

Example:
C#object obj = "hello";
if (obj is string str)   // Pattern matching (C# 7+)
    Console.WriteLine(str);

10. What is the difference between == operator and .Equals() method?

== operator: For value types, compares values. For reference types, compares references (identity) by default.
.Equals(): Virtual method. For value types, compares values. For reference types (like string), can be overridden for value equality.

Best practice: Use == for primitives, .Equals() or object.ReferenceEquals() for objects, and StringComparison for strings.

11. What is the difference between String and StringBuilder in C#?

String (System.String): Immutable. Every modification (e.g., +=, Replace, Substring) creates a new string object in memory. Good for small, fixed strings.
StringBuilder (System.Text.StringBuilder): Mutable. Designed for efficient string manipulation (especially in loops). Modifies the same object without creating new ones.

Performance Example:
C#// Bad for performance (many allocations)
string result = "";
for (int i = 0; i < 10000; i++)
    result += "a";                    // Creates 10,000+ string objects

// Good approach
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 10000; i++)
    sb.Append("a");

string final = sb.ToString();         // Only one final allocation
Rule: Use string for few operations. Use StringBuilder when doing heavy concatenation or building strings in loops.

13. What are access modifiers in C#? List and explain them.
Access modifiers control visibility of types and members:

private – Accessible only within the same class.
protected – Accessible within the same class and derived classes.
internal – Accessible within the same assembly.
protected internal – Accessible in same assembly or derived classes (even in other assemblies).
private protected (C# 7.2+) – Accessible only in same assembly and derived classes.
public – Accessible everywhere.

Best Practice: Use the most restrictive modifier possible (favor private).
14. What is the difference between static and non-static members?

static members belong to the class itself, not to any instance. Shared across all objects. Accessed via class name (ClassName.StaticMember).
non-static (instance) members belong to a specific object instance.

C#public class Counter
{
    public static int TotalCount = 0;           // Shared
    public int InstanceCount = 0;               // Per object

    public static void IncrementTotal() => TotalCount++;
}
Static members are useful for utility methods, singleton patterns, and shared state.
15. Explain the using statement and IDisposable interface.
The using statement ensures that Dispose() is called automatically when the object goes out of scope (even if an exception occurs).
C#using (var file = new StreamWriter("log.txt"))
{
    file.WriteLine("Hello");
}   // Dispose() called automatically here
Any class implementing IDisposable can be used with using. In modern C# (8+), you can also use using declarations:
C#using var file = new StreamWriter("log.txt");
This is critical for releasing unmanaged resources (files, database connections, sockets, etc.).
16. What is the difference between == and .Equals() method?

== operator:
For value types: Compares values.
For reference types: Compares references (identity) by default.

.Equals() method:
Virtual method defined in Object.
For value types and string: Compares content (value equality).
Can be overridden for custom equality logic.


Recommendation:

Use == for primitives.
Use .Equals() or implement IEquatable<T> for custom classes.
For strings, prefer string.Equals(str1, str2, StringComparison.OrdinalIgnoreCase).

Separating auto-generated and custom code.

18. Explain the difference between ref and out parameters.

ref: Parameter must be initialized before passing. Method can read and modify the value.
out: Parameter does not need to be initialized before passing. Method must assign a value before returning.

C#void RefExample(ref int x) { x = 100; }     // x must have value before call

void OutExample(out int y) { y = 200; }     // y must be assigned inside method

int a = 10;
RefExample(ref a);

int b;
OutExample(out b);
In C# 7+, you can also use out variables inline: OutExample(out int result);
19. What is method overloading and method overriding?

Method Overloading (Compile-time polymorphism):
Same method name, different parameters (number, type, or order) in the same class.

Method Overriding (Runtime polymorphism):
Child class provides specific implementation of a virtual or abstract method from the base class.
Requires virtual in base and override in derived.


Example:
C#public class Calculator
{
    public int Add(int a, int b) => a + b;           // Overloading
    public double Add(double a, double b) => a + b;  // Overloading
}

public class Animal
{
    public virtual void Speak() => Console.WriteLine("Animal sound");
}

public class Dog : Animal
{
    public override void Speak() => Console.WriteLine("Bark"); // Overriding
}
20. What is the difference between virtual, override, and new keywords?

virtual: Allows a method/property to be overridden in derived classes.
override: Overrides a virtual/abstract method in the derived class (provides polymorphic behavior).
new: Hides the base class member (method hiding). Does not participate in polymorphism.

C#public class Base
{
    public virtual void Show() => Console.WriteLine("Base");
}

public class Derived : Base
{
    public override void Show() => Console.WriteLine("Derived");   // Polymorphic
    // public new void Show() => ...   // Hides base method
}
