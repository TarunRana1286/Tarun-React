1. What is C# and what are its key features?
C# is a modern, object-oriented, type-safe programming language developed by Microsoft as part of the .NET ecosystem. It is widely used for building Windows desktop apps, web apps (ASP.NET), mobile apps (Xamarin/MAUI), cloud services (Azure), games (Unity), and more.
Key features:

Strong typing with excellent IntelliSense
Object-oriented with support for functional programming (lambdas, LINQ)
Automatic memory management via Garbage Collector
Cross-platform with .NET (formerly .NET Core)
Rich async support (async/await)
Modern features: records, pattern matching, nullable reference types, source generators, etc.
High performance with Span<T>, ref structs, and AOT compilation

3. What are the different types of data types available in C#?
C# has two main categories:

Value Types: Stored on stack (or inline), copied by value. Examples: int, float, double, bool, char, struct, enum, decimal.
Reference Types: Stored on heap, copied by reference. Examples: class, string, array, delegate, object, interface.

Also: Nullable value types (int?), dynamic, and pointer types (unsafe context).
4. What is the difference between value types and reference types?

Value Types: Contain actual data. Assignment copies the value. Passed by value to methods. Live on stack (unless boxed). Examples: primitives, structs.
Reference Types: Contain a reference (pointer) to the data on the heap. Assignment copies the reference. Passed by reference. Can be null.

Example:
C#int a = 10;          // value type
int b = a;           // b is copy of 10
b = 20;              // a remains 10

string s1 = "hello"; // reference type
string s2 = s1;      // s2 points to same string
s2 = "world";        // s1 still "hello" (strings are immutable)
5. What are nullable types in C#?
Nullable types allow value types to hold null. Introduced in C# 2.0 as Nullable<T>, shorthand T?.
C#int? age = null;
if (age.HasValue)
    Console.WriteLine(age.Value);
else
    Console.WriteLine("Age not set");

// C# 8+ Nullable Reference Types (for reference types)
string? name = null;   // Explicitly nullable
Helps prevent NullReferenceException with compiler warnings.

7. Explain the concept of boxing and unboxing in C#.

Boxing: Converting a value type to a reference type (object). Allocates memory on heap.

C#int i = 10;
object obj = i;   // boxing (performance cost)

Unboxing: Converting reference type back to value type.

C#int j = (int)obj; // unboxing
Boxing/unboxing has performance overhead. Avoid in hot paths; use generics instead.

10. What is the difference between == operator and .Equals() method?

== operator: For value types, compares values. For reference types, compares references (identity) by default.
.Equals(): Virtual method. For value types, compares values. For reference types (like string), can be overridden for value equality.

Best practice: Use == for primitives, .Equals() or object.ReferenceEquals() for objects, and StringComparison for strings.

11. What is the difference between String and StringBuilder in C#?

String (System.String): Immutable. Every modification (e.g., +=, Replace, Substring) creates a new string object in memory. Good for small, fixed strings.
StringBuilder (System.Text.StringBuilder): Mutable. Designed for efficient string manipulation (especially in loops). Modifies the same object without creating new ones.

Performance Example:
C#// Bad for performance (many allocations)
string result = "";
for (int i = 0; i < 10000; i++)
    result += "a";                    // Creates 10,000+ string objects

// Good approach
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 10000; i++)
    sb.Append("a");

string final = sb.ToString();         // Only one final allocation
Rule: Use string for few operations. Use StringBuilder when doing heavy concatenation or building strings in loops.

13. What are access modifiers in C#? List and explain them.
Access modifiers control visibility of types and members:

private – Accessible only within the same class.
protected – Accessible within the same class and derived classes.
internal – Accessible within the same assembly.
protected internal – Accessible in same assembly or derived classes (even in other assemblies).
private protected (C# 7.2+) – Accessible only in same assembly and derived classes.
public – Accessible everywhere.

Best Practice: Use the most restrictive modifier possible (favor private).
14. What is the difference between static and non-static members?

static members belong to the class itself, not to any instance. Shared across all objects. Accessed via class name (ClassName.StaticMember).
non-static (instance) members belong to a specific object instance.

C#public class Counter
{
    public static int TotalCount = 0;           // Shared
    public int InstanceCount = 0;               // Per object

    public static void IncrementTotal() => TotalCount++;
}
Static members are useful for utility methods, singleton patterns, and shared state.
15. Explain the using statement and IDisposable interface.
The using statement ensures that Dispose() is called automatically when the object goes out of scope (even if an exception occurs).

18. Explain the difference between ref and out parameters.

ref: Parameter must be initialized before passing. Method can read and modify the value.

19. What is method overloading and method overriding?

Method Overloading (Compile-time polymorphism):
Same method name, different parameters (number, type, or order) in the same class.

Method Overriding (Runtime polymorphism):
Child class provides specific implementation of a virtual or abstract method from the base class.
Requires virtual in base and override in derived.


Example:
C#public class Calculator
{
    public int Add(int a, int b) => a + b;           // Overloading
    public double Add(double a, double b) => a + b;  // Overloading
}

public class Animal
{
    public virtual void Speak() => Console.WriteLine("Animal sound");
}

public class Dog : Animal
{
    public override void Speak() => Console.WriteLine("Bark"); // Overriding
}
20. What is the difference between virtual, override, and new keywords?

virtual: Allows a method/property to be overridden in derived classes.
override: Overrides a virtual/abstract method in the derived class (provides polymorphic behavior).
new: Hides the base class member (method hiding). Does not participate in polymorphism.

C#public class Base
{
    public virtual void Show() => Console.WriteLine("Base");
}

public class Derived : Base
{
    public override void Show() => Console.WriteLine("Derived");   // Polymorphic
    // public new void Show() => ...   // Hides base method
}

22. Explain Inheritance in C# with an example.
Inheritance allows a class (derived/child) to acquire properties and methods from another class (base/parent). C# supports single class inheritance only.
C#public class Animal                  // Base class
{
    public string Name { get; set; }
    public virtual void Eat() => Console.WriteLine("Eating...");
}

public class Dog : Animal            // Derived class
{
    public void Bark() => Console.WriteLine("Woof!");

Single
Multilevel (A → B → C)

23. What is Polymorphism in C#? Explain with example.
Polymorphism means "many forms". It allows treating objects of different derived classes as objects of the base class.
There are two types:

Compile-time (Static): Method Overloading
Runtime (Dynamic): Method Overriding + Virtual methods

Example (Runtime Polymorphism):
C#public class Shape
{
    public virtual void Draw() => Console.WriteLine("Drawing shape");
}

public class Circle : Shape
{
    public override void Draw() => Console.WriteLine("Drawing Circle");
}

public class Rectangle : Shape
{
    public override void Draw() => Console.WriteLine("Drawing Rectangle");
}

// Usage
Shape shape1 = new Circle();
Shape shape2 = new Rectangle();

shape1.Draw();     // Output: Drawing Circle
shape2.Draw();     // Output: Drawing Rectangle
This is achieved through virtual method table (vtable) at runtime.

25. What is the difference between is and as operators?

30. What is the yield keyword and how does it work?
The yield keyword is used to create iterator methods that return IEnumerable<T> or IEnumerator<T>.
It allows lazy evaluation — values are generated on-demand.
C#public IEnumerable<int> GetEvenNumbers(int max)
{
    for (int i = 0; i <= max; i += 2)
    {
        yield return i;        // Returns value and pauses execution
    }
}

// Usage
foreach (var num in GetEvenNumbers(10))
    Console.WriteLine(num);
Benefits: Memory efficient for large or infinite sequences. Works beautifully with LINQ.

31. What is a Delegate in C#? Explain with an example.

Foundation for Events and LINQ.

41. What is Exception Handling in C#? Explain with keywords.
Exception handling is a mechanism to handle runtime errors gracefully so the application doesn't crash.
Key Keywords:

try – Contains code that might throw an exception.
catch – Handles the exception.
finally – Always executes (cleanup code), whether exception occurs or not.
throw – Manually throws an exception.
throws – Not used in C# (unlike Java).

Best Practice Example:
C#try
{
    int result = 10 / 0;                    // Throws DivideByZeroException
}
catch (DivideByZeroException ex)
{
    Console.WriteLine("Cannot divide by zero: " + ex.Message);
}
catch (Exception ex)                        // General catch (should be last)
{
    Console.WriteLine("Unexpected error: " + ex.Message);
}
finally
{
    Console.WriteLine("Cleanup code always runs");
}
Modern Tip: Use when clause for filtered exceptions (C# 6+):
C#catch (Exception ex) when (ex.Message.Contains("timeout"))
42. What is the difference between throw and throw ex?

throw – Re-throws the original exception while preserving the full stack trace.
throw ex – Throws the exception again but resets the stack trace (loses original call stack information).

C#try
{
    // some code that throws
}
catch (Exception ex)
{
    // Bad practice
    throw ex;                    // Stack trace is reset

    // Good practice
    throw;                       // Preserves original stack trace
}
Rule: Always use plain throw; when re-throwing in catch block.
43. What are Generics in C# and why are they useful?
Generics allow you to write reusable, type-safe code without boxing/unboxing or casting.
C#// Generic class
public class Repository<T>
{
    private List<T> _items = new List<T>();

    public void Add(T item) => _items.Add(item);
    public T Get(int index) => _items[index];
}

// Generic method
public T Max<T>(T a, T b) where T : IComparable<T>
{
    return a.CompareTo(b) > 0 ? a : b;
}
Benefits:

Type safety at compile time
Better performance (no boxing for value types)
Code reusability (List<T>, Dictionary<TKey, TValue>, etc.)
Constraints using where clause (where T : class, where T : new() etc.)

52. What is Pattern Matching in C#?
Pattern Matching allows you to test whether a value has a certain shape and extract information from it in a concise way. Introduced in C# 7 and greatly enhanced in later versions.
Examples:
C#object obj = "Hello World";

// Type Pattern (C# 7)
if (obj is string str)
    Console.WriteLine(str.ToUpper());

// Switch Expression (C# 8+ - Most Popular)
string result = obj switch
{
    string s when s.Length > 10 => "Long string",
    string s => "Short string",
    int i => $"Number: {i}",
    null => "Null value",
    _ => "Unknown type"
};

// Property Pattern
if (person is { Age: > 18, Name: not null })
    Console.WriteLine("Adult with name");
Pattern matching makes code more readable than traditional if-else or switch statements.
53. What are Nullable Reference Types? (C# 8+)
Nullable Reference Types is a feature that helps prevent NullReferenceException at compile time.
When enabled (<Nullable>enable</Nullable> in .csproj), reference types are non-nullable by default.
C#string name = null;           // Compile-time warning (non-nullable)

string? nullableName = null;  // Explicitly nullable

// Null-forgiving operator
string nonNull = nullableName!;   // Tell compiler "I know it's not null"
Best Practices:

Use string? when null is valid.
Use ! (null-forgiving) only when you are certain.
Initialize properties properly or use required init properties.

54. What is the difference between String and string?
There is no functional difference.

Convention:

Use string, int, bool (lowercase) for variable declarations and parameters (more readable).
Use String, Int32, Boolean when accessing static members (String.IsNullOrEmpty, Int32.MaxValue).

55. What is the checked and unchecked keyword?

checked

}

unchecked

58. What is the difference between IAsyncEnumerable<T> and Task<IEnumerable<T>>?

IAsyncEnumerable<T> (C# 8+): Streams data asynchronously as it becomes available (like async version of IEnumerable<T>).

C#public async IAsyncEnumerable<int> GetNumbersAsync()
{

await foreach (var number in GetNumbersAsync())
{
    Console.WriteLine(number);
}
Best Use: Streaming large datasets, real-time data, or long-running operations.

62. What is the Dispose pattern and using statement?
The Dispose pattern is used to release unmanaged resources (files, database connections, sockets, etc.).
Standard Dispose Pattern (IDisposable):
C#public class MyResource : IDisposable
{
    private bool _disposed = false;

    public void Dispose()
    {
        Dispose(true);
        GC.SuppressFinalize(this);
    }

    protected virtual void Dispose(bool disposing)
    {
        if (!_disposed)
        {
            if (disposing)
            {
                // Release managed resources
            }
            // Release unmanaged resources
            _disposed = true;
        }
    }

    ~MyResource() => Dispose(false);   // Finalizer
}
Modern C# (using declaration):
C#using var resource = new MyResource();
// resource is automatically disposed at end of scope
63. What are Expression Bodied Members?
Expression-bodied members allow writing methods, properties, and constructors using lambda-like syntax (=>).

// After C# 6+

public Person(string name) => Name = name;
Advantages: Cleaner, more concise code. Widely used in modern C#.
64. What is the difference between == and ReferenceEquals()?

== : For reference types, compares references by default (unless overloaded).
object.ReferenceEquals(): Strictly checks if two references point to the same object in memory.

C#string s1 = "hello";
string s2 = "hello";

Console.WriteLine(ReferenceEquals(s1, s2));     // May be True due to string interning
Rule: Use ReferenceEquals() when you specifically want to check object identity.
65. What is volatile keyword?
The volatile keyword tells the compiler that a field may be modified by multiple threads simultaneously. It prevents certain optimizations (caching in registers or reordering).
C#public volatile int flag = 0;
Note: volatile provides limited thread-safety. For proper thread synchronization, prefer lock, Interlocked, or concurrent collections.

67. What is the in parameter modifier? (C# 7.2+)
The in modifier passes arguments by reference without allowing modification. It is used for performance (avoids copying large structs).
C#public double CalculateDistance(in Point p1, in Point p2)
{
    // p1 and p2 cannot be modified
    return Math.Sqrt(Math.Pow(p2.X - p1.X, 2) + Math.Pow(p2.Y - p1.Y, 2));
}
Benefit: Zero-copy passing of large value types.
