1. What is SQL and what is it used for?
SQL (Structured Query Language) is the standard language for managing and manipulating relational databases. It is used to:

Create, read, update, and delete data (CRUD operations)
Define and manage database structures (tables, views, indexes)
Control access and permissions
Perform complex data analysis and reporting

It is declarative — you describe what you want, not how to do it.
2. Describe the difference between SQL and NoSQL databases.

SQL (Relational): Structured schema, uses tables with rows & columns, supports ACID transactions, JOINs, strong consistency. Best for complex queries and relationships (e.g., banking, ERP).
NoSQL: Schema-less or flexible schema, handles unstructured/semi-structured data, high scalability, eventual consistency. Best for big data, real-time apps, and high write throughput (e.g., MongoDB, Cassandra).

3. What are the different types of SQL commands?
SQL commands are categorized as:

DDL (Data Definition Language): CREATE, ALTER, DROP, TRUNCATE
DML (Data Manipulation Language): INSERT, UPDATE, DELETE, SELECT
DCL (Data Control Language): GRANT, REVOKE
TCL (Transaction Control Language): COMMIT, ROLLBACK, SAVEPOINT

4. Explain the purpose of the SELECT statement.
SELECT is the most used DML statement. It retrieves data from one or more tables.
Basic syntax:
SQLSELECT column1, column2 
FROM table_name 
WHERE condition 
ORDER BY column1;
You can use * to select all columns, but it's better to specify columns for performance.
5. What is the difference between WHERE and HAVING clauses?

WHERE: Filters rows before grouping/aggregation. Cannot use aggregate functions.
HAVING: Filters groups afterGROUP BY and aggregation. Can use aggregate functions.

SQLSELECT department, COUNT(*) as employee_count
FROM employees
WHERE salary > 50000                    -- Filters individual rows
GROUP BY department
HAVING COUNT(*) > 5;                    -- Filters groups
6. Define what a JOIN is in SQL and list its types.
A JOIN combines rows from two or more tables based on a related column.
Types:

INNER JOIN: Returns only matching rows from both tables.
LEFT JOIN (LEFT OUTER): All rows from left table + matching from right.
RIGHT JOIN (RIGHT OUTER): All rows from right table + matching from left.
FULL JOIN (FULL OUTER): All rows from both tables.
CROSS JOIN: Cartesian product (every row from first with every row from second).

7. What is a primary key in a database?
A Primary Key is a column (or set of columns) that uniquely identifies each row in a table. It must be:

Unique
Not NULL
Indexed (usually)

A table can have only one primary key.
8. Explain what a foreign key is and how it is used.
A Foreign Key is a column (or set of columns) in one table that refers to the Primary Key in another table. It enforces referential integrity.
SQLCREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    CustomerID INT,
    FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)
);
It prevents invalid data and can define cascading actions (ON DELETE CASCADE).
9. How can you prevent SQL injections?
Best ways to prevent SQL Injection:

Use parameterized queries / prepared statements (most important).
Use stored procedures.
Validate and sanitize all user input.
Use ORM (Entity Framework, Dapper) with proper parameterization.
Least privilege principle for database users.

Never concatenate user input directly into SQL strings.
10. What is normalization? Explain with examples.
Normalization is the process of organizing data to reduce redundancy and improve data integrity.
Main normal forms:

1NF: Atomic values, no repeating groups.
2NF: 1NF + no partial dependency.
3NF: 2NF + no transitive dependency.

Example: Instead of storing customer name and address in every order row, create a separate Customers table and reference it via CustomerID.

11. What are the different types of JOINs in SQL? Explain with examples.
SQL supports several types of JOINs to combine data from multiple tables:

INNER JOIN: Returns only the rows that have matching values in both tables (most commonly used).

SQLSELECT e.name, d.department_name 
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;

LEFT JOIN (LEFT OUTER JOIN): Returns all rows from the left table and matching rows from the right table. Non-matching rows get NULL.

SQLSELECT e.name, d.department_name 
FROM employees e
LEFT JOIN departments d ON e.department_id = d.id;

RIGHT JOIN (RIGHT OUTER JOIN): Opposite of LEFT JOIN — all rows from right table.
FULL OUTER JOIN: Returns all rows from both tables (matching + non-matching).
CROSS JOIN: Cartesian product — every row from first table paired with every row from second table.

Tip: Always use table aliases (e, d) for readability.
12. What is the difference between UNION and UNION ALL?

UNION: Combines results of two or more SELECT statements and removes duplicate rows. It performs sorting internally.
UNION ALL: Combines results and keeps all duplicates. Much faster because no duplicate removal or sorting.

SQL-- Removes duplicates
SELECT name FROM employees
UNION
SELECT name FROM contractors;

-- Keeps duplicates (preferred for performance)
SELECT name FROM employees
UNION ALL
SELECT name FROM contractors;
Rule: Use UNION ALL unless you specifically need to remove duplicates.
13. What is a Subquery? Explain with an example.
A Subquery (nested query) is a query inside another query. It can be used in SELECT, FROM, WHERE, or HAVING clauses.
SQL-- Find employees who earn more than the average salary
SELECT name, salary 
FROM employees 
WHERE salary > (SELECT AVG(salary) FROM employees);
Types:

Scalar Subquery (returns single value)
Row Subquery (returns one row)
Table Subquery (returns multiple rows)

Correlated Subquery: Subquery that references columns from the outer query.

15. What are Aggregate Functions? Give examples.
Aggregate functions perform a calculation on a set of values and return a single value.
Common Aggregate Functions:

COUNT() – counts rows
SUM() – total of values
AVG() – average
MAX() / MIN() – maximum / minimum value

SQLSELECT 
    department_id,
    COUNT(*) AS employee_count,
    AVG(salary) AS avg_salary,
    MAX(salary) AS max_salary
FROM employees
GROUP BY department_id;
They are usually used with GROUP BY and can be filtered using HAVING.
16. What is GROUP BY and how does it work with HAVING?
GROUP BY groups rows that have the same values in specified columns into summary rows.
HAVING filters those groups (similar to WHERE, but for aggregated data).
SQLSELECT department_id, COUNT(*) as emp_count
FROM employees
GROUP BY department_id
HAVING COUNT(*) > 10 
   AND AVG(salary) > 60000;
Order of Execution: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

18. What are Indexes in SQL? Why are they important?
An Index is a database object that improves the speed of data retrieval operations on a table.
Types:

Clustered Index: Determines physical order of data in table (only one per table, usually on Primary Key).
Non-Clustered Index: Separate structure pointing to data (multiple allowed).
Unique Index, Composite Index, Covering Index, Filtered Index.

Pros: Faster SELECT, JOIN, WHERE, ORDER BY.
Cons: Slower INSERT, UPDATE, DELETE (index needs maintenance).
Tip: Index columns used frequently in WHERE, JOIN, and ORDER BY clauses.
19. What is a View in SQL?
A View is a virtual table based on the result set of a SELECT statement. It does not store data physically (except Materialized Views).
SQLCREATE VIEW ActiveEmployees AS
SELECT id, name, department, salary 
FROM employees 
WHERE is_active = 1;
Advantages:

Simplifies complex queries
Provides security (hide sensitive columns)
Logical data independence

Note: In SQL Server/PostgreSQL, you can create Materialized Views that store data physically for better performance.
20. What is a Stored Procedure? Explain its advantages.
A Stored Procedure is a pre-compiled collection of SQL statements stored in the database.
SQLCREATE PROCEDURE GetEmployeeByDept
    @DepartmentID INT
AS
BEGIN
    SELECT name, salary 
    FROM employees 
    WHERE department_id = @DepartmentID;
END
Advantages:

Better performance (pre-compiled)
Code reusability
Security (execute permission without direct table access)
Reduced network traffic
Easier maintenance

Disadvantages: Can make debugging harder and may increase vendor lock-in.

21. What are Constraints in SQL? List and explain the main types.
Constraints are rules enforced on data columns to maintain data integrity and accuracy.
Main types of constraints:

PRIMARY KEY: Uniquely identifies each row. Cannot be NULL. Only one per table.
FOREIGN KEY: Enforces referential integrity between two tables.
UNIQUE: Ensures all values in a column are unique (can allow NULL in some databases).
NOT NULL: Prevents NULL values in a column.
CHECK: Validates that values in a column satisfy a specific condition.
DEFAULT: Provides a default value when no value is specified.

Example:
SQLCREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    Email VARCHAR(100) UNIQUE NOT NULL,
    Age INT CHECK (Age >= 18),
    DepartmentID INT,
    FOREIGN KEY (DepartmentID) REFERENCES Departments(DepartmentID)
);
22. What is a Transaction in SQL? Explain ACID properties.
A Transaction is a logical unit of work that consists of one or more SQL statements. It must be executed completely or not at all.
ACID Properties (guaranteed by relational databases):

Atomicity: All operations succeed or all fail (all-or-nothing).
Consistency: Database moves from one valid state to another.
Isolation: Transactions are isolated from each other (levels: Read Uncommitted, Read Committed, Repeatable Read, Serializable).
Durability: Once committed, changes are permanent even if the system crashes.

Basic syntax:
SQLBEGIN TRANSACTION;

UPDATE Accounts SET Balance = Balance - 1000 WHERE AccountID = 1;
UPDATE Accounts SET Balance = Balance + 1000 WHERE AccountID = 2;

COMMIT;        -- or ROLLBACK on error
23. What is the difference between COMMIT and ROLLBACK?

COMMIT: Permanently saves all changes made in the current transaction.
ROLLBACK: Undoes all changes made in the current transaction and restores the database to the state before the transaction began.

You can also use SAVEPOINT for partial rollbacks.
24. What are Triggers in SQL?
A Trigger is a special type of stored procedure that automatically executes when a specific event occurs on a table (INSERT, UPDATE, or DELETE).
Types:

AFTER Trigger – Fires after the event.
INSTEAD OF Trigger – Fires instead of the event (useful for views).

Example:
SQLCREATE TRIGGER trg_AfterInsert_Employee
ON Employees
AFTER INSERT
AS
BEGIN
    INSERT INTO AuditLog (Action, TableName, ChangedDate)
    VALUES ('INSERT', 'Employees', GETDATE());
END;
Use Cases: Auditing, enforcing complex business rules, synchronizing data.
25. What is a CTE (Common Table Expression)?
A Common Table Expression (CTE) is a temporary result set that you can reference within a SELECT, INSERT, UPDATE, or DELETE statement. It improves readability of complex queries.
SQLWITH EmployeeCTE AS (
    SELECT EmployeeID, Name, Salary, 
           ROW_NUMBER() OVER (PARTITION BY DepartmentID ORDER BY Salary DESC) as Rank
    FROM Employees
)
SELECT * FROM EmployeeCTE 
WHERE Rank = 1;        -- Get highest paid employee per department
Advantages: Better readability than subqueries, can be recursive.
26. What are Window Functions? Give examples.
Window functions perform calculations across a set of rows related to the current row (without collapsing rows like GROUP BY).
Common Window Functions:

ROW_NUMBER()
RANK(), DENSE_RANK()
LAG(), LEAD()
SUM(), AVG() with OVER()

Example:
SQLSELECT 
    Name,
    Salary,
    DepartmentID,
    ROW_NUMBER() OVER (PARTITION BY DepartmentID ORDER BY Salary DESC) as Rank,
    AVG(Salary) OVER (PARTITION BY DepartmentID) as DeptAvgSalary
FROM Employees;
Window functions are extremely powerful for analytical queries.

28. Explain Self Join with an example.
A Self Join is a regular join where a table is joined with itself. Useful for hierarchical data or comparing rows within the same table.
Example (Find employees and their managers):
SQLSELECT 
    e.name AS Employee,
    m.name AS Manager
FROM Employees e
LEFT JOIN Employees m ON e.manager_id = m.employee_id;
Common use cases: Organizational hierarchy, bill of materials, finding duplicate records.
29. What is Normalization vs Denormalization?

Normalization: Process of organizing data to minimize redundancy and dependency. Follows normal forms (1NF, 2NF, 3NF, BCNF). Improves data integrity but may require more JOINs.
Denormalization: Intentional introduction of redundancy to improve read performance. Common in data warehouses and reporting systems.

Trade-off: Normalization = better write performance & integrity. Denormalization = better read performance.

32. What is Dynamic SQL? When should you use it?
Dynamic SQL is SQL code that is generated and executed at runtime.
SQLDECLARE @sql NVARCHAR(MAX) = 'SELECT * FROM Employees WHERE DepartmentID = @DeptID';
DECLARE @DeptID INT = 5;

EXEC sp_executesql @sql, N'@DeptID INT', @DeptID = @DeptID;
Advantages: Flexibility for dynamic filters, pivoting, etc.
Disadvantages:

Risk of SQL Injection (always use parameterized queries)
Harder to debug and maintain
Can cause plan cache pollution

Best Practice: Use sp_executesql with parameters instead of plain EXEC.
33. What is a Pivot and Unpivot operation?

PIVOT: Converts rows into columns (turns category values into separate columns).
UNPIVOT: Converts columns into rows (opposite of PIVOT).

PIVOT Example:
SQLSELECT * FROM (
    SELECT DepartmentID, Salary 
    FROM Employees
) AS SourceTable
PIVOT (
    AVG(Salary)
    FOR DepartmentID IN ([1], [2], [3])
) AS PivotTable;
PIVOT is very useful for reporting and crosstab queries.
34. What are the different Isolation Levels in SQL?
Isolation levels control how transactions interact with each other:

Read Uncommitted – Can read uncommitted data (dirty reads).
Read Committed – Default in most databases. Prevents dirty reads.
Repeatable Read – Prevents non-repeatable reads.
Serializable – Highest isolation. Prevents phantom reads (most restrictive).
Snapshot (SQL Server) – Uses row versioning for better concurrency.

Trade-off: Higher isolation = better consistency but lower concurrency.
35. What is a Deadlock and how can you prevent it?
A Deadlock occurs when two or more transactions are waiting for each other to release locks, causing a standstill.
Prevention Techniques:

Keep transactions as short as possible.
Access tables in the same consistent order.
Use lower isolation levels when possible.
Avoid user interaction inside transactions.
Use NOLOCK hint carefully (for read-only queries).
Monitor with sys.dm_tran_locks or deadlock trace flags.

37. What is a Materialized View?
A Materialized View is a physical copy of the query result stored on disk (unlike a regular View which is virtual).
Advantages:

Much faster query performance for complex joins/aggregations.
Can be indexed.

Disadvantages:

Takes storage space.
Needs to be refreshed (manually or on schedule) to stay updated.

Supported in PostgreSQL, Oracle, and SQL Server (as Indexed Views).
38. What are the different types of SQL Subqueries?

Scalar Subquery: Returns a single value.
Multi-row Subquery: Returns multiple rows (used with IN, ANY, ALL).
Correlated Subquery: References columns from the outer query (executed for each row).
Nested Subquery: Subquery inside another subquery.

Example of Correlated Subquery:
SQLSELECT name 
FROM employees e
WHERE salary > (SELECT AVG(salary) 
                FROM employees 
                WHERE department_id = e.department_id);
39. How do you optimize a slow-running SQL query?
Common optimization techniques:

Add proper indexes on WHERE, JOIN, and ORDER BY columns.
Avoid SELECT * — select only required columns.
Use EXPLAIN / Execution Plan to analyze the query.
Avoid functions on indexed columns (e.g., YEAR(date_column)).
Use JOIN instead of subqueries when possible.
Limit result set with TOP / LIMIT.
Update statistics regularly.
Consider partitioning large tables.
Use covering indexes.

40. What is the MERGE statement and when is it useful?
The MERGE statement performs INSERT, UPDATE, and DELETE operations in a single statement based on a join condition.
SQLMERGE INTO TargetTable AS t
USING SourceTable AS s
ON t.ID = s.ID
WHEN MATCHED THEN
    UPDATE SET t.Name = s.Name, t.UpdatedDate = GETDATE()
WHEN NOT MATCHED BY TARGET THEN
    INSERT (ID, Name) VALUES (s.ID, s.Name)
WHEN NOT MATCHED BY SOURCE THEN
    DELETE;
Use Cases: ETL processes, data synchronization, slowly changing dimensions (SCD).

41. What is the difference between a Candidate Key and a Primary Key?

Candidate Key: Any column (or set of columns) that can uniquely identify each row in a table. It has the potential to become a primary key.
Primary Key: One chosen Candidate Key that is actually used as the unique identifier. A table can have only one Primary Key.

Example:
In a Students table:

StudentID → Candidate Key + Primary Key
Email → Candidate Key (also unique)
RollNumber → Candidate Key

A table can have multiple Candidate Keys, but only one becomes the Primary Key.
42. What is a Composite Key?
A Composite Key (or Compound Key) is a primary key made up of two or more columns combined to uniquely identify a row.
SQLCREATE TABLE OrderDetails (
    OrderID INT,
    ProductID INT,
    Quantity INT,
    PRIMARY KEY (OrderID, ProductID)   -- Composite Key
);
Use Case: When no single column can uniquely identify a record (e.g., Order + Product combination).
43. What are the different types of Relationships in a Database?
There are three main types of relationships:

One-to-One (1:1): One row in Table A relates to one row in Table B (e.g., User → UserProfile).
One-to-Many (1:N): One row in Table A relates to multiple rows in Table B (most common). Example: One Department → Many Employees.
Many-to-Many (N:N): Many rows in Table A relate to many rows in Table B. Implemented using a junction table (e.g., Students ↔ Courses via StudentCourse table).

44. What is Database Normalization? Explain 1NF, 2NF, and 3NF.
Normalization reduces data redundancy and improves data integrity.

1NF (First Normal Form): Each column contains atomic (indivisible) values. No repeating groups.
2NF (Second Normal Form): Must be in 1NF + No partial dependency (non-key columns must depend on the entire primary key).
3NF (Third Normal Form): Must be in 2NF + No transitive dependency (non-key columns must depend only on the primary key, not on other non-key columns).

Higher forms: BCNF, 4NF, 5NF (less commonly asked in interviews).
Goal: Eliminate redundancy while maintaining data integrity.
45. What is Denormalization and why is it used?
Denormalization is the intentional introduction of redundancy into a database by combining tables or adding duplicate data.
Why use it?

To improve read performance (fewer JOINs).
Common in data warehouses, reporting systems, and high-read applications.
Reduces the need for complex joins in analytical queries.

Trade-off: Better read speed but slower writes and higher risk of data inconsistency.
46. What is the COALESCE function?
COALESCE() returns the first non-NULL value from a list of arguments.
SQLSELECT 
    Name,
    COALESCE(Phone, Mobile, 'Not Available') AS ContactNumber
FROM Customers;
It is very useful for handling NULL values gracefully and providing default values.
47. What is the NULLIF function?
NULLIF(expr1, expr2) returns NULL if expr1 equals expr2, otherwise returns expr1.
Common use case: Avoid division by zero.
SQLSELECT 
    TotalSales,
    ItemsSold,
    TotalSales / NULLIF(ItemsSold, 0) AS AvgPrice
FROM Sales;

49. What are SET operators in SQL?
SET operators combine results of two or more SELECT queries:

UNION → Combines and removes duplicates
UNION ALL → Combines and keeps duplicates (faster)
INTERSECT → Returns only common rows
EXCEPT / MINUS → Returns rows from first query not present in second

All queries must have the same number of columns and compatible data types.
