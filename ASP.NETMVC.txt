1. What is ADO.NET and what are its main components?
ADO.NET is a set of managed classes in .NET for accessing and manipulating data from various data sources (databases, XML, etc.). It supports both connected and disconnected data access models.
Main components:

Data Providers: SqlClient (for SQL Server), OleDb, Odbc, etc.
Core Classes: SqlConnection, SqlCommand, SqlDataReader, SqlDataAdapter.
Disconnected Layer: DataSet, DataTable, DataView, DataRelation.

2. How does ADO.NET differ from classic ADO?
Classic ADO (pre-.NET) was COM-based, primarily connected (Recordset kept the connection open), and less scalable for web apps.
ADO.NET is fully managed, supports disconnected architecture (DataSet works offline), offers better performance, XML integration, strong typing, and no COM overhead. It is more suitable for n-tier and web applications.
3. What is the role of the DataSet in ADO.NET?
A DataSet is an in-memory, disconnected cache of data. It can hold multiple DataTables, relationships, constraints, and allows data manipulation without an active database connection. It is useful for batch updates, offline scenarios, and XML serialization.
4. Explain the differences between DataSet and DataReader.

DataReader: Forward-only, read-only, connected (connection must stay open), low memory footprint, high performance. Ideal for reading large result sets once.
DataSet: Disconnected, editable, supports multiple tables/relations, higher memory usage. Ideal when you need to modify data or navigate freely.

5. What are the key classes in ADO.NET?

SqlConnection – Manages database connection.
SqlCommand – Executes SQL/stored procedures.
SqlDataReader – Fast forward-only reader.
SqlDataAdapter – Bridges connected and disconnected models.
DataSet / DataTable – In-memory data containers.
SqlTransaction, SqlParameter, SqlBulkCopy.

6. What is the use of the Connection object in ADO.NET?
The Connection object (SqlConnection) establishes and maintains a session with the data source. Key properties/methods: ConnectionString, Open(), Close(), State. Always wrap in using for proper resource management.
7. How do you handle transactions in ADO.NET?
Use SqlTransaction:
C#using var conn = new SqlConnection(connectionString);
conn.Open();
using var transaction = conn.BeginTransaction();
try
{
    // Execute commands here with transaction passed to commands
    transaction.Commit();
}
catch
{
    transaction.Rollback();
    throw;
}
For distributed transactions, use System.Transactions.TransactionScope.
8. Describe Connection Pooling in ADO.NET and how it can be configured.
Connection Pooling reuses database connections to avoid the overhead of creating new ones. It is enabled by default.
Configure via connection string:
Pooling=true; Max Pool Size=100; Min Pool Size=5; Connection Timeout=30;
9. What is the purpose of Command objects in ADO.NET?
SqlCommand represents a SQL statement or stored procedure to execute against the database. It supports parameters, transactions, and different execution modes (ExecuteNonQuery, ExecuteReader, ExecuteScalar).
10. Can you explain what a DataAdapter does in ADO.NET?
DataAdapter bridges the connected and disconnected worlds. It fills a DataSet/DataTable using Fill() and updates the database with changes using Update(). It internally uses SelectCommand, InsertCommand, UpdateCommand, and DeleteCommand.

1–10: ASP.NET MVC Fundamentals
1. What is ASP.NET MVC and how does it differ from WebForms?
ASP.NET MVC is a web framework that implements the Model-View-Controller pattern for building scalable, testable web applications.
Key differences from WebForms:

MVC is lightweight, follows separation of concerns, supports clean URLs, and is highly testable.
WebForms is event-driven (postbacks, ViewState), heavier, and uses server controls.
MVC gives full control over HTML/HTTP; WebForms abstracts it (often resulting in large ViewState).

2. Explain the MVC architectural pattern.
MVC divides an application into three parts:

Model: Represents data and business logic.
View: Responsible for UI/presentation (Razor views).
Controller: Handles user input, orchestrates Model and View.

It promotes Separation of Concerns, testability, and maintainability.
3. What are the main components of ASP.NET MVC?

Routing (URL mapping)
Controllers & Actions
Views (Razor)
Models / ViewModels
Filters, HTML Helpers, Model Binding
Dependency Injection (in later versions)

4. How does the routing mechanism work in ASP.NET MVC?
Routing maps incoming URLs to controller actions. Configured in RouteConfig.cs (MVC 5) or Program.cs / Startup.cs (Core).
Example (conventional routing):
C#routes.MapRoute(
    name: "Default",
    url: "{controller}/{action}/{id}",
    defaults: new { controller = "Home", action = "Index", id = UrlParameter.Optional }
);
5. What is the role of the Controller in ASP.NET MVC?
The Controller receives HTTP requests, processes input, interacts with the Model, and returns an ActionResult (View, JSON, Redirect, etc.).
It acts as the orchestrator between Model and View.
6. Can you describe the lifecycle of an ASP.NET MVC request?

Request → Routing (finds route)
Controller factory creates Controller
Action filters (Authorization, Action, Result)
Action method executes
Model binding & validation
ActionResult execution (renders View)
Response sent to client

7. What are Actions in ASP.NET MVC?
Actions are public methods in a Controller that handle requests and return ActionResult. They can be decorated with attributes like [HttpGet], [HttpPost], [Authorize].
8. What is Razor View Engine?
Razor is the default view engine in ASP.NET MVC. It uses a clean, compact syntax with @ for C# code inside HTML.
Example:
HTML@model MyViewModel
<h1>@Model.Title</h1>
9. How do you pass data from a Controller to a View?

ViewBag / ViewData (dynamic, for one request)
TempData (survives one redirect)
Strongly-typed Model (recommended)
ViewModel

Example:
C#public ActionResult Index()
{
    ViewBag.Message = "Hello";
    return View(model);
}
10. What are the different ways to manage sessions in ASP.NET MVC?

Session state (HttpContext.Session)
TempData (cookie or session-based)
Cookies
Query strings / Hidden fields
Distributed cache (Redis) for scalability in Core

In modern apps, prefer stateless approaches or distributed caching.

11. Explain the concept of TempData, ViewBag, and ViewData.

ViewData: A Dictionary<string, object> that passes data from Controller to View for the current request only. Type casting is required when reading.

C#// Controller
ViewData["Message"] = "Hello World";
HTML<!-- View -->
<h2>@ViewData["Message"]</h2>

ViewBag: Dynamic wrapper around ViewData (no type casting needed). Also limited to the current request.

C#ViewBag.Message = "Hello World";   // Controller
@ViewBag.Message                    // View

TempData: Uses session (or cookies) to pass data that survives one redirect. Useful for success/error messages after POST-redirect-GET.

C#TempData["Success"] = "Record saved!";
return RedirectToAction("Index");
Key difference: ViewData/ViewBag = current request; TempData = survives one redirect. In ASP.NET Core, prefer ViewData or strongly-typed models over ViewBag for type safety.
12. What are HTML Helpers in ASP.NET MVC?
HTML Helpers are extension methods that generate HTML markup in Razor views, reducing boilerplate and ensuring consistency.
Common helpers:

@Html.TextBoxFor(model => model.Name)
@Html.LabelFor(), @Html.DropDownListFor(), @Html.ValidationMessageFor()
@Html.BeginForm(), @Html.AntiForgeryToken()
@Html.ActionLink()

Example:
HTML@Html.EditorFor(model => model.Email, new { htmlAttributes = new { @class = "form-control" } })
They support strongly-typed binding and automatic validation attributes. In modern ASP.NET Core, Tag Helpers are often preferred over HTML Helpers for cleaner HTML-like syntax.
13. How does Model Binding work in ASP.NET MVC?
Model Binding automatically maps incoming HTTP request data (form fields, query strings, route data, JSON) to action method parameters or model properties.
Process:

DefaultModelBinder looks for matching names.
Uses value providers (Form, QueryString, RouteData, etc.).
Performs type conversion and validation.

Example:
C#[HttpPost]
public ActionResult Create(UserViewModel model)  // model properties auto-populated
{
    if (!ModelState.IsValid) return View(model);
    // save
}
Custom model binders can be created by implementing IModelBinder for complex scenarios (e.g., binding to entities with relationships).
14. What is the purpose of the _ViewStart file in ASP.NET MVC?
_ViewStart.cshtml is executed before every view is rendered. It allows setting common layout or other view-wide settings.
Typical content:
HTML@{
    Layout = "~/Views/Shared/_Layout.cshtml";
}
You can add conditional logic here (e.g., different layouts based on user role). It runs for all views unless overridden in the individual view with @Layout = null; (for partial views).
15. What are Partial Views and how are they different from View Components?
Partial Views:

Reusable Razor markup (.cshtml).
Rendered using @Html.Partial("_PartialView") or @Html.RenderPartial().
Can accept a model: @Html.Partial("_LoginPartial", model).
No separate controller logic.

View Components (introduced in MVC 6 / Core):

More powerful and self-contained.
Have their own C# class (inherits ViewComponent) with Invoke() or InvokeAsync() method.
Can have logic, dependency injection, and async support.
Rendered with <vc:component-name></vc:component-name> or @await Component.InvokeAsync("ComponentName", parameters).

Difference: Partial Views are simple markup reuse. View Components are like mini-controllers for complex reusable UI (e.g., sidebar, cart summary) with business logic.
16. Explain Attribute Routing in ASP.NET MVC.
Attribute Routing allows defining routes directly on controllers and actions using attributes (introduced in MVC 5).
Advantages over conventional routing:

More precise control
Better for RESTful APIs
Supports constraints and defaults easily

Example:
C#[Route("products")]
public class ProductsController : Controller
{
    [Route("")]                    // /products
    [Route("{id:int}")]            // /products/5
    public ActionResult Details(int id) { ... }
    
    [Route("search/{term?}")]      // Optional parameter
    public ActionResult Search(string term) { ... }
}
In ASP.NET Core, attribute routing is the default/recommended approach and is more feature-rich.
17. How do you implement Forms Authentication in ASP.NET MVC?
Forms Authentication uses cookies to track authenticated users.
Steps (MVC 5):

Configure in web.config:

XML<authentication mode="Forms">
  <forms loginUrl="~/Account/Login" timeout="2880" />
</authentication>

Use [Authorize] attribute on controllers/actions.
In Login action:

C#var ticket = new FormsAuthenticationTicket(...);
var cookie = new HttpCookie(FormsAuthentication.FormsCookieName, FormsAuthentication.Encrypt(ticket));
Response.Cookies.Add(cookie);
In ASP.NET Core, Forms Authentication is replaced by Cookie Authentication + ASP.NET Core Identity (much cleaner with SignInManager and UserManager).
18. What is the AntiForgeryToken() helper and how does it work?
@Html.AntiForgeryToken() generates a hidden form field and cookie to prevent Cross-Site Request Forgery (CSRF) attacks.
How it works:

Server generates a unique token (stored in cookie + hidden field).
On POST, the framework validates that the token in the form matches the cookie token.

Usage:
HTML@using (Html.BeginForm())
{
    @Html.AntiForgeryToken()
    ...
}
On action: No extra code needed — the framework checks automatically. In Core, it's @Html.AntiForgeryToken() or the [ValidateAntiForgeryToken] attribute.
19. What is an Area in ASP.NET MVC and how is it used?
Areas allow partitioning a large application into smaller, independent sections (e.g., Admin, Blog, Shop), each with its own controllers, views, and models.
Setup:

Right-click project → Add → Area → "Admin"
Creates Areas/Admin/Controllers, Views, Models

Routing (in AreaRegistration.cs or attribute routing):
C#[Area("Admin")]
public class DashboardController : Controller { ... }
URL example: /Admin/Dashboard/Index
Areas help with organization, routing isolation, and team collaboration on large projects.
20. How do you implement error handling in ASP.NET MVC?
Multiple levels:

Action-level: Use try-catch in actions or [HandleError] attribute.
Controller-level: Override OnException in base controller.
Global-level: In FilterConfig.cs register HandleErrorAttribute.
Custom error pages: Configure in web.config:

XML<customErrors mode="On" defaultRedirect="~/Error/Error">
  <error statusCode="404" redirect="~/Error/NotFound" />
</customErrors>
In ASP.NET Core: Use middleware (app.UseExceptionHandler("/Error")) or IExceptionFilter.
Best practice: Log exceptions (using Serilog/ELMAH) and show user-friendly messages without leaking sensitive info.
