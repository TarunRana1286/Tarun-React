1. What is LINQ and why is it used?
LINQ (Language Integrated Query) is a Microsoft .NET component that adds native querying capabilities to .NET languages (C#, VB.NET). It allows you to query data from various sources (collections, databases, XML, JSON, etc.) using a consistent, SQL-like syntax directly in C#.
Benefits:

Type-safe queries
IntelliSense support
Reduced boilerplate code
Unified way to query different data sources
Deferred execution (lazy evaluation)

2. What are the two main syntaxes in LINQ?

Query Syntax (SQL-like):

C#var result = from item in collection
             where item.Age > 18
             orderby item.Name
             select item;

Method Syntax (Fluent / Lambda):

C#var result = collection
                .Where(item => item.Age > 18)
                .OrderBy(item => item.Name);
Both are compiled to the same IL. Most developers prefer Method Syntax for complex queries.
3. What is the difference between Query Syntax and Method Syntax?

Query Syntax: More readable, SQL-like. Good for joins and complex grouping.
Method Syntax: More powerful, flexible, and concise. Better for chaining operations.

Recommendation: Use Query Syntax for readability in joins/grouping, and Method Syntax for everything else. You can mix both.
4. Explain Deferred Execution in LINQ.
Deferred Execution means the LINQ query is not executed immediately when defined. It executes only when the query is enumerated (e.g., ToList(), ToArray(), foreach, or when using aggregates like Count()).
C#var query = employees.Where(e => e.Salary > 50000);   // Query is defined, not executed

// Execution happens here
var result = query.ToList();   // Deferred execution triggers now
Advantage: You can build queries dynamically and modify them before execution.
5. What is Immediate Execution in LINQ? Give examples.
Immediate Execution forces the query to run right away and materializes the result.
Common methods that cause immediate execution:

ToList(), ToArray(), ToDictionary()
Count(), Sum(), Average(), Min(), Max()
First(), Single(), Last()
Any(), All()

C#var result = employees.Where(e => e.Age > 30).ToList(); // Immediate
6. What are the main LINQ providers in .NET?

LINQ to Objects — Queries in-memory collections (List<T>, arrays, etc.)
LINQ to Entities — Queries databases via Entity Framework
LINQ to XML — Queries XML documents
LINQ to JSON — Using Newtonsoft.Json or System.Text.Json
LINQ to SQL — Legacy (replaced by EF)
PLINQ (Parallel LINQ) — Parallel execution

8. Explain the difference between First(), FirstOrDefault(), Single(), and SingleOrDefault().

First() — Returns first element. Throws exception if no element found.
FirstOrDefault() — Returns first element or default value (null for reference types).
Single() — Returns the only element. Throws if more than one or none found.
SingleOrDefault() — Returns the only element or default. Throws if more than one found.

Best Practice: Prefer FirstOrDefault() and SingleOrDefault() to avoid exceptions.
9. What is the purpose of the 'let' keyword in LINQ?
The let keyword allows you to store intermediate results in a query and reuse them.
C#var result = from e in employees
             let fullName = e.FirstName + " " + e.LastName
             where fullName.Contains("Kumar")
             select new { e.EmployeeID, FullName = fullName };
It improves readability in complex queries.
10. What are Anonymous Types in LINQ?
Anonymous types allow you to create temporary objects without defining a class.
C#var result = employees.Select(e => new 
{ 
    FullName = e.FirstName + " " + e.LastName,
    Department = e.Department.Name,
    Salary = e.Salary 
});
They are read-only and commonly used for projections in LINQ.

11. What is the purpose of the Select method in LINQ?
Select is a projection operator. It transforms each element in the source collection into a new form (similar to SELECT in SQL).
C#// Basic projection
var names = employees.Select(e => e.Name);

// Project to anonymous type
var result = employees.Select(e => new 
{ 
    FullName = e.FirstName + " " + e.LastName,
    Age = e.Age,
    Department = e.Department.Name 
});

// Index overload
var indexed = employees.Select((e, index) => new { Index = index, Employee = e });
Key Point: Select returns a new collection with transformed elements. It does not modify the original collection.
12. Explain the Where clause in LINQ.
Where is a filtering operator. It returns elements from the source collection that satisfy a given condition.
C#// Method Syntax
var seniors = employees.Where(e => e.Age >= 60);

// Query Syntax
var seniors2 = from e in employees
               where e.Age >= 60 && e.Salary > 50000
               select e;
You can chain multiple Where clauses — they are combined with AND logic.
13. What is the difference between Select and SelectMany?

Select → Projects each element to one output element (1-to-1 or 1-to-many in terms of transformation).
SelectMany → Projects each element to a collection and then flattens the result into a single collection (used for 1-to-many relationships).

C#// SelectMany example - flattening orders from multiple customers
var allOrders = customers.SelectMany(c => c.Orders);

var allOrderItems = customers.SelectMany(c => c.Orders)
                             .SelectMany(o => o.OrderItems);
Common Use: Flattening nested collections (e.g., customer → orders → order items).
14. How do you perform sorting in LINQ?
Use OrderBy, OrderByDescending, ThenBy, and ThenByDescending.
C#// Single column
var sorted = employees.OrderBy(e => e.Name);

// Multiple columns
var multiSorted = employees
                    .OrderBy(e => e.Department.Name)
                    .ThenByDescending(e => e.Salary)
                    .ThenBy(e => e.Age);
In Query Syntax:
C#var result = from e in employees
             orderby e.Department.Name, e.Salary descending
             select e;
15. Explain GroupBy in LINQ with an example.
GroupBy groups elements that share a common key.
C#// Method Syntax
var grouped = employees
                .GroupBy(e => e.Department.Name)
                .Select(g => new 
                { 
                    Department = g.Key,
                    EmployeeCount = g.Count(),
                    AvgSalary = g.Average(e => e.Salary),
                    Employees = g.ToList()
                });

// Query Syntax
var grouped2 = from e in employees
               group e by e.Department.Name into g
               select new 
               { 
                   Department = g.Key, 
                   Count = g.Count() 
               };
You can group by multiple keys using anonymous types.
16. How do you perform Joins in LINQ?
LINQ supports Inner Join, Group Join, and Left Join.
C#// Inner Join (Method Syntax)
var result = employees.Join(departments,
                e => e.DepartmentID,
                d => d.ID,
                (e, d) => new { Employee = e.Name, Department = d.Name });

// Left Join using GroupJoin + SelectMany
var leftJoin = employees.GroupJoin(departments,
                e => e.DepartmentID,
                d => d.ID,
                (e, deptGroup) => new { e, deptGroup })
                .SelectMany(
                    x => x.deptGroup.DefaultIfEmpty(),
                    (x, d) => new 
                    { 
                        Employee = x.e.Name, 
                        Department = d?.Name ?? "No Department" 
                    });
Tip: Use Query Syntax for complex joins as it is more readable.
17. What is the difference between Any() and All()?

Any() → Returns true if at least one element satisfies the condition.
All() → Returns true if every element satisfies the condition.

C#bool hasSenior = employees.Any(e => e.Age >= 60);
bool allAdults = employees.All(e => e.Age >= 18);

bool hasHighEarners = employees.Any(e => e.Salary > 100000);
Both short-circuit (stop as soon as result is known).

19. What is Take() and Skip() used for?
These are paging operators.

Take(n) → Returns first n elements.
Skip(n) → Skips first n elements.

C#// Paging: Page 3 with 10 records per page
int pageNumber = 3;
int pageSize = 10;

var pagedData = employees
                    .OrderBy(e => e.Name)
                    .Skip((pageNumber - 1) * pageSize)
                    .Take(pageSize)
                    .ToList();
Very useful for implementing pagination in web applications.
20. What is Concat() and Union() in LINQ?

Concat() → Combines two sequences keeping duplicates.
Union() → Combines two sequences and removes duplicates (like SQL UNION).

C#var combined = list1.Concat(list2);           // Keeps duplicates

var unique = list1.Union(list2);              // Removes duplicates
Union() uses Equals() and GetHashCode() for duplicate detection.

21. What is the Aggregate operator in LINQ? Explain with example.
Aggregate is a reduction operator that applies an accumulator function over a sequence and returns a single value.
C#// Simple sum using Aggregate
var sum = numbers.Aggregate((acc, x) => acc + x);

// With seed value
var totalSalary = employees.Aggregate(0m, (acc, e) => acc + e.Salary);

// Building a string
var csv = employees.Aggregate(new StringBuilder(),
    (sb, e) => sb.Append(e.Name).Append(", "),
    sb => sb.ToString().TrimEnd(',', ' '));
Common Uses: Custom sums, string concatenation, finding max/min with conditions, building complex objects.
22. What is the Zip operator in LINQ?
Zip combines two sequences element-by-element using a result selector function. It stops when the shorter sequence ends.
C#var numbers = new[] { 1, 2, 3, 4 };
var letters = new[] { "A", "B", "C" };

var result = numbers.Zip(letters, (num, letter) => $"{num}-{letter}");
// Output: "1-A", "2-B", "3-C"
Use Cases: Pairing data from two sources, merging parallel collections, creating key-value pairs.
23. Explain Distinct() in LINQ.
Distinct() returns unique elements from a sequence by removing duplicates.
C#// Simple distinct
var uniqueAges = employees.Select(e => e.Age).Distinct();

// Distinct with custom equality (using IEqualityComparer)
var uniqueEmployees = employees.Distinct(new EmployeeNameComparer());
By default, it uses the type’s Equals() and GetHashCode(). For complex types, implement IEqualityComparer<T>.
24. What are Quantifier Operators in LINQ?
Quantifier operators return a boolean value based on whether some or all elements satisfy a condition:

Any() – Returns true if at least one element matches
All() – Returns true if every element matches
Contains() – Checks if sequence contains a specific element

C#bool hasHighEarner = employees.Any(e => e.Salary > 200000);
bool allAdults = employees.All(e => e.Age >= 18);
bool hasJohn = employees.Any(e => e.Name == "John");
These operators short-circuit for better performance.

26. How do you perform Left Join in LINQ?
LINQ does not have a direct LEFT JOIN keyword. You achieve it using GroupJoin + SelectMany + DefaultIfEmpty().
C#var leftJoin = employees.GroupJoin(departments,
    e => e.DepartmentID,
    d => d.ID,
    (employee, deptGroup) => new { employee, deptGroup })
    .SelectMany(
        x => x.deptGroup.DefaultIfEmpty(),
        (x, department) => new 
        { 
            EmployeeName = x.employee.Name,
            DepartmentName = department?.Name ?? "No Department"
        });
This is one of the most frequently asked LINQ questions in interviews.
27. What is DefaultIfEmpty() used for?
DefaultIfEmpty() returns the elements of the sequence, or a default value if the sequence is empty.
It is mainly used to implement Left Joins in LINQ (as shown above).
C#var result = departments
                .Where(d => d.ID == 999)
                .DefaultIfEmpty(new Department { Name = "Unknown" });
28. Explain Union, Intersect, and Except in LINQ.
These are Set Operators:

Union → Combines two sequences and removes duplicates (like SQL UNION).
Intersect → Returns elements that appear in both sequences.
Except → Returns elements from first sequence that are not in the second sequence.

C#var union = listA.Union(listB);
var common = listA.Intersect(listB);
var onlyInA = listA.Except(listB);
All three use Equals() and GetHashCode() for comparison.
29. What is PLINQ? When should you use it?
PLINQ (Parallel LINQ) is a parallel implementation of LINQ to Objects. It automatically partitions the data and executes the query on multiple threads.
C#var result = source.AsParallel()
                   .Where(x => x.Age > 30)
                   .OrderBy(x => x.Name)
                   .ToList();
When to use PLINQ:

CPU-bound operations on large collections
When you have multiple cores available

Caution: Not suitable for I/O-bound work or when order matters (unless you use AsOrdered()).

32. What is AsEnumerable() and when should you use it?
AsEnumerable() converts an IQueryable<T> back to IEnumerable<T>. This forces the rest of the query to execute in memory instead of on the database.
C#var result = dbContext.Employees
                      .Where(e => e.Department == "IT")
                      .AsEnumerable()                    // Switch to in-memory
                      .Where(e => ComplexCalculation(e)) // Complex logic can't be translated to SQL
                      .ToList();
Use Cases:

When you need to call a method that cannot be translated to SQL.
When you want to force client-side evaluation for the remaining operations.

33. Explain AsQueryable().
AsQueryable() converts an IEnumerable<T> to IQueryable<T>. It is the opposite of AsEnumerable().
It is rarely needed but useful when you want to build a query dynamically and then execute it against a provider that supports IQueryable.
34. What are the different ways to write a Join in LINQ?
There are three main ways:

Join (Inner Join)
GroupJoin (Left Join foundation)
Query Syntax (most readable for joins)

Method Syntax (Inner Join):
C#var join = employees.Join(departments,
            e => e.DepartmentID,
            d => d.ID,
            (e, d) => new { e.Name, d.DepartmentName });
Query Syntax (Recommended for Joins):
C#var joinQuery = from e in employees
                join d in departments on e.DepartmentID equals d.ID

35. What is GroupJoin and how is it different from Join?
GroupJoin performs a left join and groups the matching elements from the right sequence.
C#var groupJoin = employees.GroupJoin(departments,
    e => e.DepartmentID,
    d => d.ID,
    (employee, deptGroup) => new 
    { 
        Employee = employee,
        Departments = deptGroup   // This is an IEnumerable
    });
It is the foundation for implementing Left Joins in LINQ.
36. How can you implement a Left Outer Join in LINQ?
The standard way is using GroupJoin + SelectMany + DefaultIfEmpty():
C#var leftOuterJoin = employees.GroupJoin(departments,
    e => e.DepartmentID,
    d => d.ID,
    (e, deptGroup) => new { e, deptGroup })
    .SelectMany(
        x => x.deptGroup.DefaultIfEmpty(),
        (x, d) => new 
        { 
            EmployeeName = x.e.Name,
            DepartmentName = d != null ? d.Name : "No Department"
        });
This is a very common interview question.
37. What is the let keyword in LINQ Query Syntax?
The let keyword introduces a new variable in the query that can be reused later.
C#var result = from e in employees
             let fullName = e.FirstName + " " + e.LastName
             let isSenior = e.Age >= 60
             where isSenior && fullName.Contains("Kumar")
             select new { FullName = fullName, IsSenior = isSenior };
It improves readability and avoids repeating expressions.
38. How do you use TakeWhile and SkipWhile?

TakeWhile → Takes elements from the start while the condition is true.
SkipWhile → Skips elements from the start while the condition is true.

C#var taken = numbers.TakeWhile(n => n < 100);     // Takes until first number >= 100

var skipped = numbers.SkipWhile(n => n < 50);    // Skips until first number >= 50
These are useful for processing ordered sequences with conditional limits.
39. What is Reverse() in LINQ?
Reverse() simply reverses the order of elements in the sequence.
C#var reversed = employees.OrderBy(e => e.Salary).Reverse();
It is often used after sorting to get descending order in a different way.
40. Explain ElementAt() and ElementAtOrDefault().

ElementAt(index) → Returns the element at the specified index. Throws exception if index is out of range.
ElementAtOrDefault(index) → Returns the element at the index or default value if out of range.

C#var thirdEmployee = employees.ElementAtOrDefault(2);   // Safer
These are useful when you need random access by index in a LINQ query result.

42. What are the Count() vs LongCount() vs Count(property)?

Count() → Returns int (32-bit). Can throw OverflowException for very large collections.
LongCount() → Returns long (64-bit). Safer for extremely large collections.
Count(predicate) → Counts only elements that match the condition.

C#int total = employees.Count();
long bigCount = employees.LongCount();

int highEarners = employees.Count(e => e.Salary > 100000);
Best Practice: Use LongCount() when dealing with potentially very large datasets.
43. What is the Sum(), Average(), Min(), and Max() in LINQ?
These are aggregate operators:
C#var totalSalary = employees.Sum(e => e.Salary);
var avgSalary   = employees.Average(e => e.Salary);
var maxSalary   = employees.Max(e => e.Salary);
var minSalary   = employees.Min(e => e.Salary);
They work with numeric types and can be used with a selector function. They cause immediate execution.
44. How do you handle NULL values in LINQ queries?
Best practices for handling NULLs:
C#// Safe navigation with null-conditional (C# 6+)
var names = employees
            .Where(e => e?.Department?.Name == "IT")
            .Select(e => e?.Name);

// Using null-coalescing
var result = employees.Select(e => new 
{ 
    Name = e.Name ?? "Unknown",
    Dept = e.Department?.Name ?? "No Department"
});

// Filter out nulls
var nonNull = employees.Where(e => e != null);
Always prefer null-conditional operators (?.) and null-coalescing (??) when working with LINQ.
45. What is OfType<T>() in LINQ?
OfType<T>() filters the collection to return only elements of the specified type (useful with non-generic collections or mixed-type sequences).
C#object[] mixed = { 1, "hello", 2.5, "world", 3 };

var onlyStrings = mixed.OfType<string>();     // Returns "hello", "world"
var onlyInts    = mixed.OfType<int>();        // Returns 1, 3
It is commonly used when working with legacy ArrayList or mixed collections.
46. Explain Cast<T>() vs OfType<T>()

Cast<T>() → Attempts to cast every element to type T. Throws InvalidCastException if any element cannot be cast.
OfType<T>() → Returns only elements that can be cast to type T. Ignores others (safer).

Recommendation: Prefer OfType<T>() unless you are sure all elements are of the target type.
47. What is SequenceEqual() in LINQ?
SequenceEqual() checks whether two sequences contain the same elements in the same order.
C#bool areEqual = list1.SequenceEqual(list2);

// With custom comparer
bool areEqualCustom = list1.SequenceEqual(list2, new EmployeeComparer());
It uses Equals() by default. Useful for unit testing and comparing collections.
48. How can you implement Paging efficiently in LINQ?
Efficient paging using Skip() and Take():
C#public async Task<PagedResult<Employee>> GetEmployeesAsync(int pageNumber, int pageSize)
{
    var totalCount = await dbContext.Employees.CountAsync();

    var items = await dbContext.Employees
                               .OrderBy(e => e.Name)
                               .Skip((pageNumber - 1) * pageSize)
                               .Take(pageSize)
                               .ToListAsync();

    return new PagedResult<Employee>
    {
        Items = items,
        TotalCount = totalCount,
        PageNumber = pageNumber,
        PageSize = pageSize
    };
}
Important: Always apply OrderBy before Skip/Take.

52. How can you improve LINQ query performance?
Key performance optimization techniques:

Use IQueryable<T> instead of IEnumerable<T> for database queries.
Avoid AsEnumerable() too early — keep query on the database as long as possible.
Select only required columns (avoid Select(e => e) or anonymous types with unnecessary data).
Use proper indexing on database columns used in Where, OrderBy, and Join.
Limit data with Take() and Skip() for pagination.
Avoid calling methods inside Where that cannot be translated to SQL.
Use AsNoTracking() in EF Core for read-only queries.
Materialize early (ToList()) only when necessary.

53. What is AsNoTracking() in Entity Framework Core?
AsNoTracking() tells EF Core not to track the returned entities. This improves performance significantly for read-only queries.
C#var employees = await dbContext.Employees
                               .AsNoTracking()
                               .Where(e => e.DepartmentID == 5)
                               .ToListAsync();
When to use:

Reporting queries
Read-only APIs
Large result sets

It reduces memory usage and increases speed because EF doesn't maintain change tracking.

62. How does Distinct() work with custom objects?

C#public class EmployeeNameComparer : IEqualityComparer<Employee>
{
    public bool Equals(Employee x, Employee y) => x?.Name == y?.Name;
    public int GetHashCode(Employee obj) => obj.Name?.GetHashCode() ?? 0;
}

// Usage

This is a common interview follow-up question.
63. What is the difference between Concat() and Union()?

Concat() → Combines two sequences as-is, keeping duplicates.
Union() → Combines two sequences and removes duplicates (uses Equals + GetHashCode).

C#var all = listA.Concat(listB);      // Keeps duplicates
var unique = listA.Union(listB);    // Removes duplicates
Rule: Use Concat() when you want to preserve duplicates; use Union() when you want unique elements.
64. Explain Except() and Intersect() operators.

Except() → Returns elements from the first sequence that are not in the second sequence.
Intersect() → Returns elements that are present in both sequences.

C#var onlyInA = listA.Except(listB);
var common   = listA.Intersect(listB);
Both support custom IEqualityComparer<T> for complex types.
65. What are Extension Methods in the context of LINQ?
All standard LINQ operators (Where, Select, OrderBy, etc.) are implemented as Extension Methods on IEnumerable<T> and IQueryable<T>.
C#public static class MyLinqExtensions
{
    public static IEnumerable<T> WhereNotNull<T>(this IEnumerable<T> source)
    {
        return source.Where(x => x != null);
    }
}

// Usage
var cleanList = employees.WhereNotNull();
This is how you can create your own reusable LINQ methods.
66. Can LINQ perform recursive queries? How?
LINQ itself does not support direct recursion, but you can achieve recursive behavior using a recursive method or CTEs (in SQL via EF).
Example with a recursive method:
C#public IEnumerable<Employee> GetAllSubordinates(Employee manager)
{
    var direct = employees.Where(e => e.ManagerId == manager.Id);
    return direct.Concat(direct.SelectMany(e => GetAllSubordinates(e)));
}
For database-level recursion, use Recursive CTEs in SQL and map via Entity Framework.

71. What is the difference between Select and SelectMany? (Deep Dive)
This is one of the most important LINQ questions asked in interviews.

Select → Transforms each element into one output element (1-to-1 projection).
SelectMany → Transforms each element into a collection, then flattens all those collections into a single sequence (handles 1-to-many relationships).

C#// Select - Returns collection of collections
var selectResult = customers.Select(c => c.Orders);           // IEnumerable<IEnumerable<Order>>

// SelectMany - Flattens into single collection
var selectManyResult = customers.SelectMany(c => c.Orders);   // IEnumerable<Order>
Real-world Use: Flattening nested structures like Customer → Orders → OrderItems.
72. How can you implement a Left Outer Join using LINQ Method Syntax?
LINQ does not have a built-in LEFT JOIN. You achieve it using GroupJoin + SelectMany + DefaultIfEmpty():
C#var leftJoin = employees.GroupJoin(departments,
    e => e.DepartmentID,
    d => d.ID,
    (employee, deptGroup) => new { employee, deptGroup })
    .SelectMany(
        x => x.deptGroup.DefaultIfEmpty(),
        (x, department) => new 
        { 
            EmployeeName = x.employee.Name,
            DepartmentName = department?.Name ?? "No Department"
        });
This is a classic interview question. Make sure you can explain every part.
73. What is the into keyword in LINQ Query Syntax?
The into keyword is used to continue a query after a group by or join (it creates a temporary identifier for the result).
C#var result = from e in employees
             join d in departments on e.DepartmentID equals d.ID into empDept
             from ed in empDept.DefaultIfEmpty()   // Left Join
             select new { e.Name, Department = ed?.Name ?? "None" };
It is also used after group by to continue the query.
74. What is Deferred Execution vs Immediate Execution? Give examples of both.

Deferred Execution: The query is not executed when defined. It runs only when enumerated.C#var query = employees.Where(e => e.Salary > 50000);   // Not executed yet
var result = query.ToList();                          // Executes here
Immediate Execution: The query runs immediately.C#var count = employees.Count(e => e.Age > 30);         // Immediate
var list = employees.ToList();                        // Immediate

Most LINQ operators are deferred except aggregators (Count, Sum, Average, etc.) and materialization methods (ToList, ToArray, ToDictionary).
75. Can you use LINQ with async/await? How?
Yes. In modern .NET, use Entity Framework Core async methods:
C#var result = await dbContext.Employees

76. What is IAsyncEnumerable<T> and how does it relate to LINQ?
IAsyncEnumerable<T> is the async version of IEnumerable<T>. It allows streaming data asynchronously.
C#public async IAsyncEnumerable<Employee> GetEmployeesAsync()
{
    await foreach (var employee in dbContext.Employees.AsAsyncEnumerable())
    {
        // Process asynchronously
        await Task.Delay(10);
        yield return employee;
    }
}
You can use LINQ-like methods with System.Linq.Async package for IAsyncEnumerable.
77. How do you write a recursive query using LINQ?
LINQ does not support recursion natively. Common approaches:

Recursive Method:

C#public IEnumerable<Employee> GetHierarchy(Employee manager)
{
    yield return manager;
    foreach (var child in employees.Where(e => e.ManagerId == manager.Id))
        foreach (var descendant in GetHierarchy(child))
            yield return descendant;
}

For database: Use Recursive CTE in SQL and map via EF Core.

78. What is the OfType<T>() operator used for?
OfType<T>() filters a collection and returns only elements that can be cast to the specified type T.
C#object[] mixed = { 1, "hello", 2.5, "world", true, 42 };

var strings = mixed.OfType<string>();   // "hello", "world"
var numbers = mixed.OfType<int>();      // 1, 42
It is very useful when working with non-generic collections (ArrayList, object[], etc.).
79. Explain the Cast<T>() operator.
Cast<T>() attempts to cast every element in the sequence to type T. It throws InvalidCastException if any element cannot be cast.
C#object[] objects = { 1, 2, 3 };
var numbers = objects.Cast<int>();     // Works

// If one element fails → exception
Difference from OfType<T>(): Cast is strict, OfType is safe (filters out incompatible elements).
80. What are some common LINQ anti-patterns / mistakes to avoid?
Common mistakes:

Calling ToList() too early → loses deferred execution benefits.
Using AsEnumerable() unnecessarily on database queries.
Using complex methods inside Where that cannot be translated to SQL.
Not using AsNoTracking() for read-only EF queries.
Ignoring query execution plans.
Using foreach with Add instead of Select + ToList().
Overusing anonymous types in public APIs.

Best Practice: Keep queries as IQueryable<T> as long as possible.
